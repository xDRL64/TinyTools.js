By clicking on the feature "Display args in console", the menu generated with SCCM will call the function 'content()' of this feature by passing it one argument.
In this example we just display it with console.log() .
By reading the log in console we can see that the argument is an Object, containing the following properties :

- allowedEventProps, Type : {buttons: number, shiftKey: boolean, ctrlKey: boolean, altKey: boolean}
- contextData,       Type : Any
- forced,            Type : Boolean
- item,              Type : HTMLElement

The property allowedEventProps is present if 'content()' function was been called from a "standard click" of SCCM, which can be triggered by simple click on a any feature.
"standard click" can be disabled, see the precise doc pages about menu settings (out of "quick start" section).
If it is present, this property contains an Object which contains some js nativ MouseEvent's properties directly transfered from internal SCCM js Event.
It provides : mouse buttons' state as bit flags in a "1byte value", and keyboard flags for CTRL, SHIFT, ALT as booleans.

The property contextData is always present and its state depends on you. If you do not set it, it will be 'null'.
It is the most important data used when we click on a feature beause in major cases it is supposed to be the data repsenting the right-clicked element calling our context menu, and so, to be short it is the main data we wish interact with when we are clicking on a feature.
To set it, use handle[ukey].set_contextData(data) just after a right click but before opening/dispaying the context menu.
To reset it, use handle[ukey].remove_contextData(), or handle[ukey].set_contextData(null) which are totaly equivalent.
